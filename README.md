## 🇬🇧 English

This project extends **GLarena**, the viewer used in the Graphics course at the FIB, by integrating **Godot** as a complementary environment for shader experimentation.

The main contributions of this project are:

- Automatic scene generation in Godot.
- Translation of **GLSL** shaders into **GDShader** by traversing the Abstract Syntax Tree (AST) generated by **glslang**.
- Translation tables based on JSON files.
- Automatic UV coordinate generation for models that do not include them.
- A preconfigured visualization environment with multiple cameras and lighting.

The goal of this project is to provide students with a more visual and interactive way to learn and experiment with shaders, introduce them to the **Godot** game engine, and promote the use of *open-source* software.

The `viewer` directory contains the original **GLarena** documentation corresponding to the repository version downloaded on **February 24, 2026**. It also includes the original contents of the **glslang** repository, corresponding to the revision used on the same date, together with its README and the documentation provided by its authors.

This document describes the dependencies required to build and run the project, as well as the instructons of the new features added. For additional information, please refer to the official documentation of the original projects.

#### Official Resources

- Khronos Group — https://www.khronos.org
- Virvig — https://www.virvig.eu

### Requirements

> **Note:** This project was developed on **Ubuntu 24.04 LTS**. Although this is the reference platform, other Linux distributions can also be used, provided that the dependencies listed below are available.

- Linux operating system.
- Qt 5.4 or later.
- C++17-compatible compiler (GCC or Clang).
- Godot 4.5 or later with OpenGL 3.3 support.

### Using the New Features

Once the **viewer** has been started (for more information, see the `README.md` located in the `newviewer/` directory), the new functionality can be accessed by pressing the **Space** key and opening the **Shaders** menu.

The following options are available:

- **New VS + FS in Godot...**
- **Export VS + FS to Godot...**

#### New VS + FS in Godot...

Generates a default shader named `DefaultShader.gdshader` and automatically assigns it to the default model loaded in the scene.

#### Export VS + FS to Godot...

Translates a *Vertex Shader* and a *Fragment Shader* into a `.gdshader` file with the same name, which is saved in:

```text
viewer/godot/GDshaders/
```

Both options generate and save all the required resources in the `DefaultScene.tscn` scene, located in:

```text
viewer/godot/
```

Once the process is complete, the project is automatically opened in **Godot** with the generated scene loaded.

## 🇪🇸 Español

Este proyecto amplía **GLarena**, el visor utilizado en la asignatura de Gráficos de la FIB, integrando **Godot** como un entorno complementario para la experimentación con shaders.

Las principales aportaciones del proyecto son:

- Generación automática de escenas en Godot.
- Traducción de shaders de **GLSL** a **GDShader** mediante el recorrido del Árbol de Sintaxis Abstracta (AST) generado por **glslang**.
- Uso de tablas de traducción basadas en archivos JSON.
- Generación automática de coordenadas UV para modelos que no disponen de ellas.
- Entorno de visualización preconfigurado con múltiples cámaras e iluminación.

El objetivo es proporcionar a los estudiantes una forma más visual e interactiva de aprender y experimentar con shaders, introducirlos al motor de videojuegos **Godot** y fomentar el uso de software *open source*.

Dentro del directorio `viewer` se incluyen las instrucciones originales de **GLarena** correspondientes a la versión del repositorio descargada el **24 de febrero de 2026**. Asimismo, se incorpora el contenido original del repositorio **glslang**, correspondiente a la revisión realizada en esa misma fecha, junto con su README y la documentación de uso proporcionada por sus autores.

En este documento se describen las dependencias necesarias para compilar y ejecutar el proyecto, así como las instrucciones sobre las nuevas funciones añadidas. Para obtener información adicional, se recomienda consultar la documentación oficial de los proyectos originales.

#### Recursos oficiales

- Khronos Group — https://www.khronos.org
- Virvig — https://www.virvig.eu

### Requisitos

> **Nota:** El proyecto fue desarrollado sobre **Ubuntu 24.04 LTS**. Aunque esta es la plataforma de referencia, es posible utilizar otras distribuciones de Linux siempre que dispongan de las dependencias indicadas a continuación.

- Sistema operativo Linux.
- Qt 5.4 o superior.
- Compilador compatible con C++17 (GCC o Clang).
- Godot 4.5 o superior con soporte para OpenGL 3.3.

### Uso de las nuevas implementaciones

Una vez iniciado el **viewer** (para más información, consulta el `README.md` situado en el directorio `newviewer/`), las nuevas funcionalidades estarán disponibles al pulsar la **barra espaciadora** y acceder al menú **Shaders**.

Las nuevas opciones son:

- **New VS + FS in Godot...**
- **Export VS + FS to Godot...**

#### New VS + FS in Godot...

Genera un shader por defecto denominado `DefaultShader.gdshader` y lo asigna automáticamente al modelo cargado por defecto en la escena.

#### Export VS + FS to Godot...

Traduce un *Vertex Shader* y un *Fragment Shader* a un archivo `.gdshader` con el mismo nombre, que se guarda en el directorio:

```text
viewer/godot/GDshaders/
```
Ambas opciones generan y guardan todos los recursos necesarios en la escena DefaultScene.tscn, ubicada en:
```text
viewer/godot/
```
Una vez finalizado el proceso, el proyecto se abre automáticamente en Godot con dicha escena cargada.
